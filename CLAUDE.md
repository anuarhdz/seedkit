# CLAUDE.md

## Project

**seedkit** — pnpm monorepo of CLI tools for generating and transforming MDX demo/seed data for local development.

- npm: `seedkit`
- GitHub: https://github.com/anuarhdz/seedkit
- Author: Anuar Reyes

## Packages

| Package              | Path              | Purpose                                             |
| -------------------- | ----------------- | --------------------------------------------------- |
| `@seedkit/core`      | `packages/core`   | Shared types, serializers, file writer, Zod schemas |
| `@seedkit/generate`  | `tools/generate`  | Generate fake MDX files from a schema via Faker     |
| `@seedkit/scrape`    | `tools/scrape`    | Scrape web pages → MDX (crawl or sitemap discovery) |
| `@seedkit/transform` | `tools/transform` | Batch-transform MDX frontmatter/metadata fields     |

## Key commands

```bash
pnpm generate       # run @seedkit/generate (requires generate.config.ts in cwd)
pnpm scrape         # run @seedkit/scrape (requires scrape.config.ts in cwd)
pnpm transform      # run @seedkit/transform (requires transform.config.ts in cwd)
pnpm typecheck      # typecheck all packages
pnpm lint           # lint all files
pnpm release        # release-it interactive (bumps version, generates changelog, tags)
pnpm release:patch  # non-interactive patch release
```

## Architecture

- Tools are not libraries — they run from the user's cwd and read a local config file
- Config files (`generate.config.ts`, etc.) are TypeScript, loaded via dynamic `import()` and validated with Zod
- `@seedkit/core` is a workspace-only package (`workspace:*`), not published to npm
- All tools use `tsx` at runtime — no build step required for development
- Output format can be `frontmatter` (YAML) or `metadata-export` (`export const metadata = {}`)

## Conventions

- Commits follow Conventional Commits — enforced by Commitlint + Husky
- Changelog is auto-generated by `release-it` + `@release-it/conventional-changelog`
- Prettier: `singleQuote: false` (double quotes)
- TypeScript: strict mode, NodeNext resolution
- No publishing individual tool packages to npm — only root `seedkit` is reserved on npm as a placeholder
